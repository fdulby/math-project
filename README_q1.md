# 迪士尼乐园路线优化系统 - 使用说明

## 文件说明

1. **q1.py** - 主程序文件
2. **projects_data.csv** - 项目数据文件

## 功能特点

### 1. 数据驱动
- 从CSV文件读取项目信息
- 支持自定义距离矩阵

### 2. 多人群支持
- 普通游客：平衡各类项目
- 亲子游：偏好互动、休闲项目
- 情侣游：偏好刺激、沉浸项目

### 3. 多日期类型
- 工作日：排队时间×0.7
- 双休日：排队时间×1.0
- 节假日：排队时间×1.5

### 4. 改进的目标函数
```
综合得分 = 总效用 - 时间成本 - 超时惩罚 - 错过演出惩罚 + 多样性奖励

其中：
- 时间成本 = 0.1×排队 + 0.15×等待 + 0.05×步行
- 超时惩罚 = 5×超时²
- 错过演出惩罚 = 1000×错过场次
- 多样性奖励 = 2×项目类型数
```

### 5. 模拟退火优化
- 初始温度：1000
- 降温速率：0.995
- 最大迭代：5000次

## CSV文件格式

```csv
项目ID,项目名称,是否演出,时间窗开始,时间窗结束,游玩时长,刺激度,沉浸度,互动度,休闲度,基础排队,峰值1强度,峰值1时间,峰值1宽度,峰值2强度,峰值2时间,峰值2宽度
1,创极速光轮,否,,,5,0.9,0.5,0.1,0.1,15,60,120,30,50,420,40
4,奇梦幻影秀,是,360,370,20,0.2,0.9,0.5,0.6,,,,,,
```

### 字段说明
- **项目ID**: 唯一标识（对应距离矩阵索引）
- **是否演出**: "是"或"否"
- **时间窗开始/结束**: 演出项目的时间窗（分钟，以开园为0）
- **游玩时长**: 项目游玩时间（分钟）
- **刺激度/沉浸度/互动度/休闲度**: 0-1之间的特征值
- **基础排队**: 平峰期最少排队时间（分钟）
- **峰值参数**: 高斯峰 (强度, 时间, 宽度)

## 距离矩阵格式

n×n矩阵，其中：
- 第0行/列：入口节点
- 第1-n行/列：各项目节点
- 单位：分钟（步行时间）

示例（7×7矩阵）：
```python
distance_matrix = np.array([
    [0, 10, 15, 12, 8, 20, 18],   # 从入口到各项目
    [10, 0, 8, 10, 12, 15, 20],   # 项目1到其他
    ...
])
```

## 使用方法

### 方法1：直接运行
```bash
python q1.py
```

### 方法2：自定义参数
修改`main()`函数中的参数：
- `selected_crowd`: 人群类型
- `selected_date`: 日期类型
- `distance_matrix`: 距离矩阵
- `park_close_time`: 闭园时间（分钟）

### 方法3：作为模块导入
```python
from q1 import load_projects_from_csv, calculate_utility_scores, simulated_annealing

# 读取数据
project_info, df = load_projects_from_csv('projects_data.csv')

# 计算效用
project_info = calculate_utility_scores(project_info, '亲子')

# 运行优化
best_route, best_result = simulated_annealing(
    project_ids=[1, 2, 3, 4, 5, 6],
    distance_matrix=your_matrix,
    project_info=project_info,
    park_close_time=720
)
```

## 输出结果

### 1. 综合指标
- 综合得分
- 总效用
- 总耗时
- 排队/等待/步行时间
- 错过演出数
- 超时情况
- 可行性

### 2. 推荐路线
按顺序列出项目名称

### 3. 详细时间线
每个项目的到达、排队、等待、离开时间和效用得分

## 注意事项

1. **项目ID必须连续**：从1开始，对应距离矩阵的索引
2. **时间单位统一**：所有时间均为分钟
3. **演出时间窗合理**：避免过晚导致长时间等待
4. **距离矩阵对称**：通常步行时间是双向对称的
5. **特征值归一化**：刺激度等特征值应在0-1之间
